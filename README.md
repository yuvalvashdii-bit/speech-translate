# 🎙️🌍 תרגום חי בשידור — Live Speech Translation

הדובר מדבר **בעברית**, והצופים רואים את התרגום **בזמן אמת** בשפה שהם בוחרים (ברירת מחדל: **רוסית**).
הדובר שולח קישור אחד — כל מי שפותח אותו בוחר שפה ורואה כתוביות חיות.

## 💸 חינם לגמרי — בלי מפתח, בלי חיוב

- **זיהוי הדיבור** מתבצע בדפדפן של הדובר (Web Speech API) — חינם ובזמן אמת.
- **התרגום** דרך שירות חינמי (Google Translate / MyMemory) — בלי מפתח API.
- עלות כוללת: **0 ₪**.

> **דרישה אחת:** הדובר חייב לדבר דרך **Google Chrome על מחשב** או על **טלפון אנדרואיד**.
> ב-Safari באייפון זיהוי הדיבור בדפדפן לא נתמך. (הצופים — על כל מכשיר שהוא, הם רק מקבלים טקסט.)

## איך זה עובד

```
דובר (Chrome) → זיהוי דיבור עברית בדפדפן (חינם) → טקסט → שרת
                                    → תרגום חינמי לכל שפת צופה → צופים (כתוביות חיות)
```

## פריסה לענן (מומלץ — קישור קבוע + HTTPS)

> מיקרופון בדפדפן דורש **HTTPS**. פריסה לענן נותנת HTTPS אוטומטית.

### Render (הכי פשוט)

1. העלה את התיקייה הזו ל-GitHub (repo חדש).
2. ב-[Render](https://render.com) → **New → Blueprint** → בחר את ה-repo (יש `render.yaml`).
3. Deploy. **לא צריך להגדיר שום מפתח** — זה עובד חינם מהקופסה.
4. תקבל כתובת כמו `https://speech-translate-xxxx.onrender.com`.

חלופות זהות: **Railway**, **Fly.io** — כל פלטפורמת Node עם WebSocket. פקודת התחלה `npm start`.

## הרצה מקומית (לבדיקה)

```bash
npm install
npm start
# פתח http://localhost:3000  ב-Chrome
```
בדיקה מהמחשב עצמו עובדת (localhost נחשב מאובטח). לבדיקה מטלפון אנדרואיד צריך HTTPS —
פרוס לענן, או `cloudflared tunnel --url http://localhost:3000`.

## שימוש

1. **הדובר** (ב-Chrome) פותח את האתר → **"התחל שידור חדש"**.
2. **מעתיק את קישור הצופים** ושולח (וואטסאפ/מייל).
3. לוחץ **"התחל לדבר"**, מאשר מיקרופון, ומדבר בעברית.
4. **כל צופה** פותח את הקישור, **בוחר שפה** (רוסית כברירת מחדל), ורואה תרגום חי.
   אפשר לסמן "הצג מקור (עברית)".

## מילון מונחים (חינם, תמיד פעיל)

שמות החברה מוגנים אוטומטית כך שיישארו **בדיוק** בכל שפה (גם ברוסית):
`אצמ` / `א.צ.מ` → **ATM**, ו-`סטירלינק`/וריאציות → **Steerlinq**. גם מתקן שגיאות זיהוי נפוצות.
לעריכה/הוספת מונחים — הרשימה `GLOSSARY` בראש `server.js`.

## שדרוג איכות (אופציונלי, בתשלום — מומלץ לאירוע המנכ"ל)

ברירת המחדל (Google חינמי + מילון) טובה מאוד. לאיכות ורישום (register) הכי מדויקים —
מנוע **Claude** שמתרגם כל משפט גמור *בידיעה* שזה מנכ"ל שמדבר לעובדים על תוצאות רבעון ב-ATM ו-Steerlinq.
התצוגה החיה תוך כדי דיבור נשארת על המנוע החינמי המהיר; רק השורה הננעלת עוברת שדרוג — כך שזה מהיר וזול.

| משתנה | ברירת מחדל | תיאור |
|---|---|---|
| `TRANSLATION_ENGINE` | `free` | `free` / `claude` / `openai` / `deepl` |
| `ANTHROPIC_API_KEY` | — | נדרש ל-`claude` (חשבון עם קרדיט; עלות לאירוע = סנטים בודדים) |
| `CLAUDE_MODEL` | `claude-haiku-4-5` | מודל מהיר וזול, מתאים ללייב |
| `TALK_CONTEXT` | (מוגדר מראש) | ההקשר שנמסר למנוע — אפשר לעדכן לכל אירוע |
| `OPENAI_API_KEY` | — | נדרש ל-`openai` |
| `DEEPL_API_KEY` | — | נדרש ל-`deepl` |
| `SOURCE_LANG` | `he` | שפת הדובר |
| `PORT` | `3000` | פורט |

## הערות ומגבלות

- **דובר על אייפון:** לא נתמך במסלול החינמי (אין זיהוי דיבור ב-Safari). דרוש Chrome (מחשב/אנדרואיד). אם *חייבים* אייפון — צריך זיהוי דיבור בצד שרת (בתשלום, למשל OpenAI עם קרדיט).
- **תרגום חינמי:** ה-endpoint של Google הוא לא-רשמי; אם ייכשל, המערכת נופלת אוטומטית ל-MyMemory. לשימוש כבד/מסחרי מתמשך עדיף מנוע בתשלום.
- **השהיה:** בדרך כלל 1–2 שניות אחרי סוף משפט (זיהוי בדפדפן מהיר; רק התרגום עובר בשרת).
- **מופע יחיד:** הסשנים בזיכרון. לריצה במספר מכונות צריך אחסון משותף (Redis).
